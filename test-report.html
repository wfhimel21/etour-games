‚úÖ Escalation System Fixes (January 1, 2026)

All 4 failing tests fixed - escalation system now fully operational!

Root Cause Identified: The matchLevel3Delay configuration was incorrectly set to 2 minutes (identical to L2 delay), causing L2 and L3 escalation levels to become available simultaneously. This violated the intended escalation progression where each level should activate sequentially with proper spacing. Additionally, test files were using hardcoded timeout values instead of reading the actual contract configuration, leading to test failures when configuration was updated.

Fix Applied: Updated matchLevel3Delay from 2 minutes to 4 minutes (cumulative from initial timeout) across all four ConnectFour tiers. This creates proper escalation spacing: L1 becomes available immediately at timeout (5 minutes), L2 becomes available 2 minutes later (7 minutes total), and L3 becomes available 2 minutes after that (9 minutes total). Test files were refactored to dynamically read timeout configuration from the contract rather than using hardcoded values, making tests resilient to future configuration changes.
